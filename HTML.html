HTML Básico

  1. Estructura de una Etiqueta

    hay dos tipos de etiquetas en html, las que se tienen que cerrar y las que no; ejemplos

    • Se cierran

      • como con los encabezados: "< h1 >Titulo < / h1>"

    • No se cierran

      • como las imagenes: "< img src="" >

  2. Párrafos y Encabezados

    para escribir un párrafo se usa la etiqueta "p".

    para los encabezados hay 6 posiblidades, h1, h2, h3, h4, h5 ,h6; la diferencia esta en la importancia del titulo, siendo h1 el título principal y que se recomienda usar uno solo en toda la página; y desde el h2 en adelante se usan como subtitulos, pero siguiendo un orden

    Ejemplos:

    < p >párrafo< /p >

    < h1 >Título< /h1 >

  3. Listas

    Hay dos tipos de listas: Ordenadas y Desordenadas

    Para usar una lista ordenada se usa la etiqueta "ol" y para usar una lista desordenada se usa "ul"; adentro de estas etiquetas su usan los "li" para separar un item de otro

    Ejemplos:

    < ol >
    _< li >< /li >
    _< li >< /li >
    _< li >< /li >
    < /ol >

    < ul >
    _< li >< /li >
    _< li >< /li >
    _< li >< /li >
    < /ul >

  4. Enlaces

    Para usar un enlace en html se necesita de la etiqueta "a", ademas es muy importante un atributo para enlazar la dirección, el atributo es "href", va dentro de la etiqueta

    Ejemplo:

    < a href="dirección" >ENLACE 1< /a >

    los enlaces por naturaleza se abren o cargan en la misma página, es decir, al hacer click en el la dirección o página se carga en la misma pestaña

    ¿Cómo cambiar para que se carguen en otra pestaña?

    Para esto, hay que modificar el atributo "target", que por defecto su valor es "_self", hay que cambiarlo a "_blank"

    Ejemplo:

    < a href="enlace" target="_blank" >ENLACE 2< /a >

  5. Imagenes

    la etiqueta para la imagenes es "img", y es una etiqueta que no se cierra; para que funcione se necesita el atributo "src" que sirve para alojar la dirección de la imagen para que así se pueda mostrar; ademas es recomendable usar el atributo "alt" que sirve para dejar un mensaje en caso de que la imagen no se logre mostrar, un mensaje como de que trata la supuesta imagen

    Ejemplo:

    < img src="direccion de la imagen" alt="imagen de ejemplo">

  6. Rutas

    Hay dos tipos de Rutas: Rutas Absolutas y Rutas Relativas

    Rutas Absolutas

    Para entenderlas mejor, son rutas que sea donde sea van a llegar al mismo lugar; esto como los enlaces en la web

    Ejemplo:

    "https://youtube.com"

    Rutas Relativas

    Este tipo de ruta dependen de la ubicación del archivos al que estamos llamando.

    Si se encuentra en la misma carpeta, basta con colocar el nombre del archivo

    Ejemplo: "imagen.jpg"

    Si se encuentra una carpetamas allá, o en una carpeta que se encuentra en la misma carpeta del archivo donde estamos se debe usar el nombre de la carpeta donde está el archivo que buscamos y luego si el nombre del archivo

    Ejemplo: Imagenes/foto.png
    asi con las carpetas que sean necesarias.

    Si el acrchivo que buscamos está afuera de la carpeta donde estamos se debe usar "../" para salir una carpeta, si fuesen dos habría que repetir lo mismo

    Ejemplo: ../imagen.png

    si fuersen dos carpetas afuera: ../../archivo.jpg

    Tambien podemos entrar a otra carpeta despues de salir una

    Ejemplo: ../imagenes/portada.png
    en ese ejemplo sale una carpeta y luego ingresa a otra carpeta donde está el archivo que se necesita

  7. Formularios

    Para trabajar con formularios es necesario utilizar la etiqueta "form", dentro de esta se trabaja cada elemento que hace parte del formulario

    La etiqueta que se trabaja dentro de "form" principalmente es "input"

    Input

    El imput representa un elemento que participa en el formulario, como un espacio para escribir texto, como el nombre, como un email; un imput puede ser tambien tipo numérico, de color; puede ser de tipo archivo, que permite subir algun archivo, tambien esta el input submit que es el boton que envía el formulario

    Atributos en un input
    1. type: es donde se define el tipo de input, como de texto, email, numerico etc
    2. placeholder: es un atributo que permite agregar un texto en el input como de ejemplo
    3. Required: es un atributo que hacer que el input sea obligatorio de llenar para poder enviarse
    4. Name: sirve como un identificador, es util para funcionalidades en la parte del backend
    5. Value: es el contenido que se envia en cada input, ejemplo, el nombre escrito por el usuario etc
    6. min y max: en algunos inputs se puede aplicar para definir el mínimo o máximo de caracteres que deben ser rellenados, por ejemplo el nombre mínimo 3 caracteres
